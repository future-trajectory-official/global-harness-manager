# Technical Spec: Git Protection for Published Rules

グローバルハーネスから配布された共有ルールが、ターゲットプロジェクトの Git リポジトリに誤って混入することを防ぐための自動化仕様です。

## 1. 自動ディレクトリ生成
同期先プロジェクトに `.agents/rules` が存在しない場合、スクリプトが自動的にディレクトリを作成します。

## 2. 動的な `.gitignore` 更新
ルールが同期される際、ターゲットの `.agents/rules/.gitignore` に対して、配布されたファイル名が自動的に追記されます。
- **重複排除**: すでに登録されている場合は二重に追記しません。
- **効果**: これにより、ユーザーやエージェントがターゲットプロジェクトで `git add .` を実行しても、共有ルールは Git の管理対象外（Untracked）のまま維持されます。

## 3. 安全性の意図
共有ルールは `global-harness-manager` で一元管理されるべきものであり、各プロジェクトで個別に変更してコミットすることを防ぐのが目的です。
