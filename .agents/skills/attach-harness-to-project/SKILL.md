# attach-harness-to-project

既存のリポジトリに対して、グローバルハーネスで管理されている Git アイデンティティ（ユーザー名・メールアドレス）を装着し、通信経路（Remote URL）を専用の SSH エイリアスに強制書き換えを行います。

## 主な機能
1. **リモート URL の変換**: `https://github.com/` や `git@github.com:` の形式を、`git@github.com-[本人エイリアス]:` の形式に書き換えます。
2. **ローカル Git 設定の固定**: 指定したプロジェクト内限定で `user.name` と `user.email` を `identities.txt` に基づいて自動設定します。
3. **誤操作防止**: 対象ディレクトリが有効な Git リポジトリでない場合はエラーを返します。

## 使用方法
以下の引数を与えて実行スクリプトを呼び出してください。

### 実行スクリプト
- `scripts/harness-attach.sh <アカウント名> <プロジェクトディレクトリへの絶対パス>`

## 安全性への配慮
- 設定が既に存在する場合、不要な書き換えはスキップします。
- `identities.txt` にアカウント情報が存在しない場合は実行を中断し、不整合を防ぎます。

## 推奨ワークフロー
Antigravity が新しい既存プロジェクトの管理を任された際、最初にこのスキルを実行することで、以降の git commit や push が適切な権限で、かつ正しいメタデータを持つことを保証できます。
