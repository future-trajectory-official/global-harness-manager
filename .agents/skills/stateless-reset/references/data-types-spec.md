# Technical Spec: Reset Target Directories

`stateless-reset` が退避対象とする主要なディレクトリの定義と役割です。

## 1. `brain/`
- **内容**: エージェントが過去のタスクや対話から生成した推論プロセス、中間ログ、および学習データ。
- **リセットの必要性**: 過去の試行錯誤がノイズとなり、現在のタスクにおいて最適な判断を妨げる可能性があるため。

## 2. `knowledge/`
- **内容**: 特定のトピックやリポジトリに関して抽出・要約された「知識アイテム (KI)」。
- **リセットの必要性**: コードベースの大幅な変更やブランチの切り替えにより、保持している知識が古くなった（Staleになった）場合にリフレッシュするため。

## 3. `conversations/`
- **内容**: 過去のチャットセッションの履歴データ。
- **リセットの必要性**: 会話の文脈が長くなりすぎ、エージェントの短期記憶（Context Window）を圧迫するのを避けるため。

## 4. 退避の仕組み
物理的な「削除」ではなく、システムの一時ディレクトリ（`/tmp` 等）への「移動 (mv)」を行います。これにより、不測の事態（誤って重要な記憶を消してしまった場合）に、一定期間内であれば手動で復元することが可能です。
