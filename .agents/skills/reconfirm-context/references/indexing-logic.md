# Technical Spec: Skill Indexing Logic

エージェントが利用可能なスキル（Capability）を再認識するためのインデックス生成の仕組みです。

## 1. スキルパスの探索

本スキルは、`~/.gemini/antigravity/skills.txt`
に登録されたディレクトリを基点に、再帰的に探索を行います。

```bash
$(cat ~/.gemini/antigravity/skills.txt | xargs -I {} find {} -maxdepth 3 -name index-capabilities.sh | head -1)
```

- **`skills.txt`**: ユーザーによって許可された「スキル配置ルート」のリスト。
- **`index-capabilities.sh`**:
  各スキルセットのルートに配置される、目次（Manifest）生成用のスクリプト。

## 2. インデックスの役割

生成されたインデックスは、エージェントに対して「今、何ができるのか（どのディレクトリにどんな
SKILL.md
があるのか）」を明示します。これにより、エージェントは過去の会話履歴に頼ることなく、現在の実態に基づいたツール選択が可能になります。

## 3. コンテキストの最適化

インデックス化の最大の目的は、大量のファイルの中身をすべて読み込むのではなく、概要（メタデータ）のみを読み込むことで、トークン消費を抑え、エージェントの推論精度を維持することにあります。
